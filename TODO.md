# 今後のTODOリスト（熱盛プロジェクト）

現在のシステムはローカル（ご自身のPC上）で基本機能が完成し、スプレッドシートとの連携が完了した状態です。
今後、このシステムを社内で完全に稼働させるために必要となるステップをまとめています。

## 1. AI連携の準備（Gemini APIキーの取得）
現在、メール解析機能の裏側は準備できていますが、実際にAIを動かすための「カギ」が必要です。
- [x] [Google AI Studio](https://aistudio.google.com/) にアクセスし、Googleアカウントでログインする
- [x] 「Get API key」または「Create API key」から新しいAPIキーを発行する
- [x] 発行したキーをコピーし、プロジェクト直下の `.env.local` ファイルの `GEMINI_API_KEY=` の右側に貼り付ける

## 2. サーバーへのデプロイ（Webへの公開）
メールの自動受信システム（Webhook）を動かすには、アプリをPCの中だけでなく、インターネット上に置く必要があります。（※社内専用にするための鍵かけも可能です）
- [x] **Vercel** または **Google Cloud Run** 等のアカウントを作成する
- [x] 現在のプロジェクトを GitHub 等にプッシュし、Vercelと連携してデプロイする
- [x] デプロイ先の環境変数設定（Environment Variables）に、`.env.local` に書いた3つの情報（メールアドレス、秘密鍵、シートID、Geminiキー）をすべて登録する

## 3. メール受信システム（Webhook）の設定
専用のメールアドレス（例: `inbound@atsumori.com`）を用意し、そこに届いたメールをアプリに転送する設定です。
- [x] **SendGrid** などのメール配信サービスでアカウントを作成する（※今回はより簡単なMakeを使用）
- [x] Inbound Parse（受信フック）の機能設定画面を開く
- [x] 転送先のURL（Destination URL）として、デプロイしたアプリのURL（例: `https://atsumori-app.vercel.app/api/webhooks/inbound-email`）を登録する

## 3.5 MakeからGmail (GAS) への移行とテスト
Make.comの動作不安定を解消するため、専用GmailアカウントとGASを用いた直接転送に切り替える。
- [x] 熱盛用のGmailアカウントにGASの転送スクリプト（`forwardToVercelWebhook`）を配置する
- [ ] スマホ等から添付ファイル付きのテストメールを送信し、VercelのWebhookが正常にデータを受け取って処理するかテストする
- [ ] 問題なくスプレッドシートに書き込まれることを確認したら、GASで1分間隔の定期実行トリガーを設定する
